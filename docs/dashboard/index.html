Architecture at a glance

Four-stage pipeline with explicit fail-closed gates between each: download → section detection → per-trial extraction → cross-register dedup → versioned CSV. Three specific corruption-mode guards (negated counts, chart-axis numbers, questionnaire-acronym false positives) encoded as regression tests. Phase 3 semantic HR scoring applied to FDA narrative extraction only.

Verdict policy: every extracted row is compared by hand against the trial's published NEJM primary. "Clean" requires exact match on HR, both CI bounds, and N. "Noisy" rows are shipped in the CSV but flagged with this verdict and should not be used for downstream analysis without re-extraction.

See methods-paper.md for architecture details and the honest audit of what this pipeline does and doesn't do.
